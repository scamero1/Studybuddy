Documento de Transparencia sobre Uso de IA – StudyBuddy

Resumen
Este documento describe el uso de herramientas de IA durante el desarrollo y dentro de la aplicación, las modificaciones humanas realizadas y cómo se verificó el correcto funcionamiento. El repositorio actual contiene una aplicación de escritorio Java (`Swing`) y pruebas básicas.

Qué parte fue generada/asistida por IA
- Diseño de clases y estructura POO: `studybuddy.StudyBuddy` con clases internas `Note`, `User`, `UserStore`, `NoteStore`, `PublicStore` y `StudyBuddyFrame` (java/studybuddy/StudyBuddy.java:1–991).
- Lógica de resumen y palabras clave basada en frecuencia de términos: métodos `summarize`, `keywords`, `termFreq`, `scoreSentence` (java/studybuddy/StudyBuddy.java:284–360).
- Interfaz gráfica `Swing`: disposición de paneles, acciones de botones y tema (java/studybuddy/StudyBuddy.java:669–990).
- Integración opcional con Groq API: `AiService` y `GroqAiService`, prompts y cliente HTTP (java/studybuddy/StudyBuddy.java:576–668).
- Pruebas básicas: `StudyBuddyTests` con validaciones de hash, roundtrip JSON y funciones (java/studybuddy/StudyBuddyTests.java:1–35).

Qué modificaciones humanas se hicieron
- Ajustes de textos y mensajes de interfaz para claridad.
- Revisión de rutas de datos y compatibilidad Windows.
- Inclusión/edición de README y organización de instrucciones.
- Limpieza y verificación manual de flujos: login/registro, CRUD, públicas, IA.

Cómo comprobar que funciona correctamente
- Compilación y ejecución:
  - `javac -d java/bin java/studybuddy/StudyBuddy.java`
  - `java -cp java/bin studybuddy.StudyBuddy`
- Pruebas automatizadas básicas:
  - `javac -d java/bin java/studybuddy/StudyBuddy.java java/studybuddy/StudyBuddyTests.java`
  - `java -cp java/bin studybuddy.StudyBuddyTests`
- Pruebas manuales guiadas:
  - Crear/editar/eliminar notas; marcar como públicas; ver públicas.
  - Probar resumen, palabras clave y ejercicios con y sin `groq.key`.
  - Verificar que `data/users.json`, `data/notes_<usuario>.json` y `data/public_notes.json` se actualizan.

Alcance y limitaciones
- IA opcional: si no hay clave en `groq.key` o rutas soportadas, las funciones de IA retornan vacío y la UI lo comunica.
- Persistencia local en JSON; no apto para producción sin servidor y controles adicionales.
- Algoritmos de resumen/keywords simples basados en TF; suficientes para nivel principiante.
